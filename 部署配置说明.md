# 四叶草情侣 App 部署配置说明

## 一、需要修改的文件

### 1. 后端配置文件
**文件路径：** `myphd/sever/src/main/resources/application.properties`

**需要修改的内容：**

```properties
# 服务器端口（保持 8080 或改为其他端口）
server.port=8080

# 数据库连接（改为云服务器的 MySQL 配置）
spring.datasource.url=jdbc:mysql://localhost:3306/couples_app?useSSL=true&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
spring.datasource.username=你的MySQL用户名
spring.datasource.password=你的MySQL密码

# 文件上传路径（改为云服务器的路径）
file.upload-dir=/home/ubuntu/couples_app/uploads
# 或者使用：file.upload-dir=/var/www/couples_app/uploads

# 文件访问 URL（改为你的服务器 IP 或域名）
file.access-url=http://你的服务器IP:8080/uploads/
# 如果有域名：file.access-url=https://你的域名.com/uploads/
```

### 2. Android 客户端 API 地址
**文件路径：** `myp/app/src/main/java/com/example/myp/data/api/RetrofitClient.kt`

**当前配置：**
```kotlin
private const val BASE_URL = "http://192.168.31.185:8080/api/"
```

**需要修改为：**
```kotlin
// 方式1：使用服务器 IP
private const val BASE_URL = "http://你的服务器IP:8080/api/"

// 方式2：使用域名（推荐）
private const val BASE_URL = "https://你的域名.com/api/"
```

## 二、云服务器环境准备

### 1. 安装 Java 17
```bash
sudo apt update
sudo apt install openjdk-17-jdk -y
java -version  # 验证安装
```

### 2. 安装 MySQL
```bash
sudo apt install mysql-server -y
sudo mysql_secure_installation  # 配置安全设置
```

### 3. 创建数据库
```bash
sudo mysql -u root -p

# 在 MySQL 中执行：
CREATE DATABASE couples_app CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'couples_user'@'localhost' IDENTIFIED BY '你的密码';
GRANT ALL PRIVILEGES ON couples_app.* TO 'couples_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

### 4. 创建上传目录
```bash
sudo mkdir -p /home/ubuntu/couples_app/uploads
sudo chmod 755 /home/ubuntu/couples_app/uploads
```

## 三、部署步骤

### 步骤 1：修改配置文件
1. 修改 `application.properties`（数据库、上传路径、访问 URL）
2. 修改 `RetrofitClient.kt`（API 地址）

### 步骤 2：打包后端 JAR
```bash
cd myphd/sever
./mvnw clean package -DskipTests
# JAR 文件生成在：target/sever-0.0.1-SNAPSHOT.jar
```

### 步骤 3：上传 JAR 到服务器
```bash
# 在本地执行（替换为你的服务器 IP）
scp target/sever-0.0.1-SNAPSHOT.jar root@你的服务器IP:/home/ubuntu/couples_app/
```

### 步骤 4：在服务器上运行
```bash
# SSH 登录服务器
ssh root@你的服务器IP

# 进入目录
cd /home/ubuntu/couples_app

# 运行应用（后台运行）
nohup java -jar sever-0.0.1-SNAPSHOT.jar > app.log 2>&1 &

# 查看日志
tail -f app.log
```

### 步骤 5：重新构建 Android APK
```bash
cd myp
./gradlew assembleRelease
# APK 生成在：app/build/outputs/apk/release/app-release.apk
```

### 步骤 6：更新服务器上的 APK
```bash
# 复制新 APK 到服务器静态资源目录
cp app/build/outputs/apk/release/app-release.apk ../myphd/sever/src/main/resources/static/
# 然后重新打包后端并上传
```

## 四、配置域名和 HTTPS（可选但推荐）

### 1. 购买域名
- 阿里云、腾讯云、GoDaddy 等

### 2. 域名解析
在域名控制台添加 A 记录：
- 主机记录：`@` 或 `www`
- 记录值：你的服务器 IP

### 3. 安装 Nginx
```bash
sudo apt install nginx -y
```

### 4. 配置 Nginx 反向代理
创建配置文件 `/etc/nginx/sites-available/couples_app`：
```nginx
server {
    listen 80;
    server_name 你的域名.com;

    location /api/ {
        proxy_pass http://localhost:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /uploads/ {
        proxy_pass http://localhost:8080/uploads/;
    }

    location / {
        proxy_pass http://localhost:8080/;
    }
}
```

启用配置：
```bash
sudo ln -s /etc/nginx/sites-available/couples_app /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

### 5. 配置 HTTPS（使用 Let's Encrypt 免费证书）
```bash
sudo apt install certbot python3-certbot-nginx -y
sudo certbot --nginx -d 你的域名.com
```

## 五、配置自动启动（systemd）

创建服务文件 `/etc/systemd/system/couples-app.service`：
```ini
[Unit]
Description=Couples App Spring Boot Service
After=syslog.target network.target

[Service]
User=ubuntu
ExecStart=/usr/bin/java -jar /home/ubuntu/couples_app/sever-0.0.1-SNAPSHOT.jar
SuccessExitStatus=143
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启用服务：
```bash
sudo systemctl daemon-reload
sudo systemctl enable couples-app
sudo systemctl start couples-app
sudo systemctl status couples-app
```

## 六、防火墙配置

### Ubuntu UFW
```bash
sudo ufw allow 8080/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
```

### 云服务器控制台
在阿里云/腾讯云安全组中开放端口：
- 8080（Spring Boot）
- 80（HTTP）
- 443（HTTPS）

## 九、安全建议

2. **禁用 root SSH 登录**

4. **开启防火墙**
5. **定期备份数据库和上传文件**
6. **使用 HTTPS（必须）**
